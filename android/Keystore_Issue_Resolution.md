# Android 키스토어 서명 오류 해결 가이드

## 🚨 현재 상황
Google Play Console에서 키 서명 오류가 발생했습니다:

### 요구되는 키 지문:
```
SHA1: 0D:D6:90:A2:0B:92:0B:20:93:0B:C6:E1:71:D9:02:40:64:B1:10:9B
```

### 현재 업로드된 키 지문:
```
SHA1: 78:5B:5C:8C:53:84:A1:18:39:FD:D1:85:6B:E8:C7:4C:EA:49:D4:C0
```

## 해결 방법

### 방법 1: Google Play Console에서 새 앱 만들기 (권장)

1. **Google Play Console** 접속
2. **모든 앱** → **앱 만들기** 클릭
3. **새 앱 정보 입력**:
   ```
   앱 이름: YCITY+
   기본 언어: 한국어 (대한민국)
   앱 또는 게임: 앱
   무료 또는 유료: 무료
   ```
4. **앱 만들기** 및 **새 AAB 업로드**

### 방법 2: 기존 키스토어로 서명 (복잡함)

기존 앱에서 요구하는 키스토어를 찾아서 사용해야 합니다.

**⚠️ 주의**: 기존 키스토어가 없다면 이 방법은 불가능합니다.

## 권장 해결책: 새 앱 등록

### 1. 현재 AAB 파일 정보
- **위치**: `build/app/outputs/bundle/release/app-release.aab`
- **크기**: 26.6MB
- **키 지문**: `78:5B:5C:8C:53:84:A1:18:39:FD:D1:85:6B:E8:C7:4C:EA:49:D4:C0`
- **패키지명**: `com.ilsan_ycity.ilsanycityplus`

### 2. 새 앱 등록 절차
1. **Google Play Console** → **새 앱 만들기**
2. **기본 정보 입력**
3. **현재 AAB 파일 업로드** (키 지문이 자동으로 등록됨)
4. **스토어 등록 정보 작성**
5. **출시**

### 3. 앱 메타데이터
이미 준비된 메타데이터 사용:
- **파일**: `android/Google_Play_Store_Metadata.md`
- **앱 설명**: ✅ 완료
- **스크린샷**: 촬영 필요
- **개인정보 처리방침**: 작성 필요
